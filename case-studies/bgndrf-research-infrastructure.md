# Case Study: BGNDRF Research Infrastructure Modernization

## Role

IT Manager / IT Director

## Dates

2007 to 2010

## Context

When I joined BGNDRF in Alamogordo, New Mexico, the organization supported research projects using four physically separate networks. Each network had its own hardware and file server, which allowed four independent research projects to operate at the same time.

The facility itself had capacity for significantly more concurrent work, but the network design limited utilization.

## Challenge

The existing environment provided isolation, but it did so through rigid physical separation. That created unnecessary hardware overhead, limited scalability, and restricted the number of projects the facility could support.

## Solution

I rebuilt the network from the ground up and implemented segmented infrastructure that preserved project isolation while allowing the facility to use shared compute resources more efficiently.

Key work included:

- Redesigning the network architecture.
- Implementing segmented network infrastructure.
- Building a four-node VMware ESXi cluster.
- Creating isolated resources for each network segment.
- Enabling more concurrent research projects across the facility.

## Results

The new architecture allowed the organization to support more concurrent projects while maintaining separation between research environments. It also reduced physical hardware dependency and introduced a repeatable model for provisioning isolated project infrastructure.

## Career Impact

This was one of my earliest major infrastructure modernization projects. It shaped how I think about virtualization, segmentation, multi-tenant infrastructure, and designing systems around mission capability rather than hardware boundaries.
